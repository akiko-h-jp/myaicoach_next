# デプロイ前チェックリスト

GitHubにプッシュしてVercelにデプロイする前に、以下の項目を確認してください。

## ✅ セキュリティ確認

- [ ] `.env`ファイルが`.gitignore`に含まれているか確認
- [ ] `.env.local`ファイルが`.gitignore`に含まれているか確認
- [ ] `.env.bak`ファイルが`.gitignore`に含まれているか確認
- [ ] `.env.example`ファイルが存在し、実際の値が含まれていないか確認
- [ ] `git status`で`.env`ファイルが表示されないことを確認

## ✅ ファイルの準備

- [ ] `.env.example`ファイルが存在する
- [ ] `README.md`にデプロイ手順が記載されている
- [ ] `package.json`に`postinstall`スクリプトが設定されている

## ✅ コードの確認

- [ ] ローカルで`npm run build`が成功する
- [ ] ローカルで`npm run dev`が正常に動作する
- [ ] すべての機能が正常に動作する

## ✅ GitHubへのプッシュ

- [ ] `git status`でコミットするファイルを確認
- [ ] `.env`ファイルが含まれていないことを確認
- [ ] コミットメッセージを記入
- [ ] GitHubにプッシュ

## ✅ Vercelでの設定

- [ ] Vercelアカウントでログイン
- [ ] GitHubリポジトリをインポート
- [ ] すべての環境変数を設定
- [ ] `NEXTAUTH_URL`を本番環境のURLに設定
- [ ] デプロイを実行
- [ ] デプロイが成功することを確認

## ✅ デプロイ後の確認

- [ ] デプロイされたURLにアクセスできる
- [ ] ログイン機能が正常に動作する
- [ ] タスク管理機能が正常に動作する
- [ ] スケジューリング機能が正常に動作する
- [ ] LINE通知機能が正常に動作する（設定している場合）

## トラブルシューティング

### ビルドエラーが発生する場合

1. ローカルで`npm run build`を実行してエラーを確認
2. エラーメッセージを確認して修正
3. Vercelのビルドログを確認

### 環境変数が読み込まれない場合

1. Vercelダッシュボードで環境変数が正しく設定されているか確認
2. 環境変数名にタイプミスがないか確認
3. 環境変数の値に余分なスペースがないか確認
4. 再デプロイを実行

### データベース接続エラーが発生する場合

1. `DATABASE_URL`が正しいか確認
2. Supabaseのデータベースが起動しているか確認
3. ファイアウォール設定を確認
